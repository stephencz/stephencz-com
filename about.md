#### Jekyll
I made this website using the [static site](https://en.wikipedia.org/wiki/Static_web_page){:target="_blank"} generator [Jekyll](https://jekyllrb.com/){:target="_blank"}.

Before static site generators, maintaining anything more than a few static web pages was impractical.
Truly static websites serve their pages as they are stored.
They do not rely on server-side programs to dynamically generate content as it is requested.

In the past, if you wanted to change an attribute common to all pages of a static website, such as the header, footer, navigation, or general layout, then you had to go to each page and make the change manually.
This was not good or fun.
Dynamic websites, such as those that run on [WordPress](https://wordpress.com/){:target="_blank"}, have the advantage of server-side languages.
Using PHP a Wordpress website can maintain each component of the final web page in seperate HTML files.
There is a file for the header, a file for the navigation, a file for the footer, and a file for the body.
Then, when a page is requested, it is dynamically pieced together, and served to your web browser. 

Jekyll uses a combination of languages, most notably [Ruby](https://www.ruby-lang.org/en/){:target="_blank"}, [Liquid](https://shopify.github.io/liquid/){:target="_blank"}, and [YAML](https://en.wikipedia.org/wiki/YAML){:target="_blank"}, to partially close the gap between dynamic and static websites.
